# CSC313 Weather Dashboard with D3

Your assignment is to create a weather dashboard that includes these elements:

- Geolocation
- Using HTML sliders to set arguments
- Using a web API to fetch data
- Using D3 to display the data

Your initial website will look like this:

<img width="481" height="374" alt="2026-03-29 21_44_21-313Weather1" src="https://github.com/user-attachments/assets/6b4d0bdb-bb04-4202-be5c-7dfb5c3fd4ed" />

The button will use the standard browser API to get your location (latitude and longitude), and then will query the open-meteo API to get data. The sliders control how many days of past data and forecast data to retrieve. The result will look something like this (you are allowed to use your own creativity to make this look better!)

<img width="386" height="622" alt="2026-03-29 21_45_19-313Weather2" src="https://github.com/user-attachments/assets/384cd75d-1d6a-4bc2-9444-d76f72c6cce6" />

As you can see the weather data is labeled with annotations, and the time frame for data is indicated. The "Available" indicates that location services worked, and returned the latitude and longitude that are shown.

When you run this, you will get a pop-up asking if the site can use your location. You should say yes ... or it will fail.

If you accidentally say no, you can go to your settings and reset the option for this "website".

In Chrome, Google says to look under Permissions at Settings > Privacy and security > Site settings > Location
Safari is similar, Safari > Settings > Websites > Location

You are starting from scratch, with no HTML or Javascript template.

YOU MUST PUT YOUR NAME IN THE SUBMSSION! This is critical, as there are some mixed up userid to Github ID matchings.
